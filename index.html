GF-based speech recognition grammars for Estonian

Introduction

Our grammars cover some fixed domains (e.g. calculator, maps) and support two languages Est and App for talking about these domains.

  • Est corresponds to spoken Estonian and is meant to guide the speech recognizer (i.e. it is compiled into JSGF)
  • App corresponds to a machine language (e.g. standard calculator, Google Maps query language)

GF is a grammar formalism with a focus on translation. Describing the domains and languages in GF allows us to translate from Est to App in order to "execute" an Estonian utterance. For example, a spoken input (in Est):

Algus Keskuse kaheksa teist Lõpp Filtri tee Tallinn
(Begin Keskuse eighteen End Filtri street Tallinn)

could be translated into Google Maps query language (App)

FROM Keskuse 18 , Tallinn TO Filtri tee , Tallinn

which can be directly executed by resolving the URL http://maps.google.com/maps?q=FROM+Keskuse+18+,+Tallinn+TO+Filtri+tee+,+Tallinn.

Grammars

The grammars can be downloaded in 2 "compiled" formats:

  • jsgf: Est-grammar in JSGF format (plain text), which can be used for speech recognition
  • pgf: both grammars packaged in a binary format (can be opened by GF), which can be used for translation

The GF source code is available at http://github.com/Kaljurand/Grammars/.

Examples

GF commandline example

Translation is performed by parsing as Est and linearizing the resulting syntax tree into App.

$ gf Direction.pgf
Direction> parse -lang=DirectionEst "Algus Keskuse kaheksa teist Lõpp Filtri tee Tallinn" | linearize -lang=DirectionApp
FROM Keskuse 1 8 , Tallinn TO Filtri tee , Tallinn
